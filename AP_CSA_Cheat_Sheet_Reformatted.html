Formula bank and memorization targets

These are the patterns students most often need to recall quickly.

Random integer in a closed interval

Generates a random integer from a through b, inclusive.

(int)(Math.random() * (b - a + 1)) + a

Cast before dividing

Integer division truncates. Convert first when you need decimal output.

(double)5 / 2 // 2.5

Positive rounding trick

Works for positive values only.

(int)(x + 0.5)

Range check

Write a bounded range as a compound boolean expression.

lo <= x && x <= hi

Null check first

Let short-circuit evaluation prevent a NullPointerException.

obj != null && obj.method()

Safe removal in an ArrayList

When removing elements, traverse from the end toward the start.

for (int i = list.size() - 1; i >= 0; i--)

Binary search loop rule

The loop continues while the search interval is still valid.

while (lo <= hi)

Neighbor access in a 2D array

Check bounds before accessing adjacent cells, and use && so the second test only runs when safe.

r > 0 && grid[r - 1][c] == target
Unit 1A • 15-25%

Primitives and expressions

Foundational syntax and evaluation rules. Students need this material for both tracing and writing code.

Data types and variables

Primitives: int, double, boolean. These store values directly.

Reference types: String, arrays, and objects. These store memory addresses.

Declaration examples: int x = 5;, double y = 3.0;, boolean b = true;

Constants: final int MAX = 100; cannot be reassigned.

Default values for fields: int → 0, double → 0.0, boolean → false, reference types → null.

Local variables do not get default values.

Arithmetic and casting
Operator / ideaExampleWhat to remember
+, -, *3 * 2.0 → 6.0int with double gives a double
/7 / 2 → 3int / int truncates
%7 % 2 → 1Sign follows the dividend
(int)(int)3.9 → 3Truncates, does not round
(double)(double)5 / 2 → 2.5Cast before divide

Overflow: Integer.MAX_VALUE + 1 wraps to Integer.MIN_VALUE. No runtime error.

Precedence: () → unary (-, !) → *, /, % → +, - → relational → && → ||

Assignment and output

Compound assignment: +=, -=, *=, /=, %=

Increment and decrement: x++ and x--. The exam only uses postfix form.

String concatenation: "hi" + 5 → "hi5"

Left-to-right evaluation: 1 + 2 + "a" → "3a", but "a" + 1 + 2 → "a12"

Printing: System.out.print() does not add a newline. System.out.println() does.

Most-tested reminders
Common trap int / int always truncates: 5 / 3 → 1, -7 / 2 → -3.
Study tip When a quotient should be decimal, cast first: (double)5 / 3 → 1.667
Exam habit On multiple choice, add mental parentheses and trace the expression in precedence order.
Unit 1B

Objects, methods, and strings

This section combines object construction, method calls, and the String API.

Objects and constructors

Create an object: ClassName obj = new ClassName(args);

new allocates memory and calls the constructor.

null: an uninitialized reference. Calling a method on null causes a NullPointerException.

Overloaded constructors: same name, different parameter lists. The compiler chooses by argument count and types.

No-argument constructor: ClassName(). Java supplies one only when you have written no constructors at all.

Methods and Math

Instance method: obj.method(args)

Static method: ClassName.method(args)

Void methods perform an action and return nothing.

Non-void methods return a value that must be stored or used.

Common Math methods: Math.abs(x), Math.pow(b, e), Math.sqrt(x), Math.random()

Random decimal range: Math.random() gives a value in [0.0, 1.0).

Method signature: access modifier + return type + method name + parameters. Example: public int getAge()

String methods (Strings are immutable)
MethodReturnsMeaning
.length()intNumber of characters. Includes parentheses.
.substring(a, b)StringCharacters from index a to b - 1
.substring(a)StringFrom index a to the end
.indexOf(s)intReturns -1 if not found

Immutability: String methods return a new String. The original is unchanged unless you reassign it, as in s = s.toUpperCase();

Most-tested reminders
Common trap == compares String references. Use .equals() to compare content.
Study tip Students should be able to read API entries by checking the return type, method name, and parameter list in order.
Memorize The inclusive random integer formula is a high-frequency AP CSA pattern.
Unit 2A • 25-35%

Boolean logic and conditionals

Students need to evaluate conditions correctly and understand how boolean logic controls program flow.

Boolean expressions

Relational operators: ==, !=, <, >, <=, >= produce boolean values.

Logical operators: &&, ||, !

Short-circuit rules: && stops at the first false, and || stops at the first true.

De Morgan's laws:

  • !(A && B) ≡ !A || !B
  • !(A || B) ≡ !A && !B

Range check: lo <= x && x <= hi

if / else if / else

Structure: if (cond) { ... } else if (cond2) { ... } else { ... }

Only one block runs.

Nested if: the inner if runs only when the outer condition is true.

Dangling else: an else matches the nearest unmatched if. Braces help avoid mistakes.

Equivalent logic: if (!cond) A else B is equivalent to if (cond) B else A.

Comparing objects

Primitives: == compares values.

Objects and Strings: use .equals() for content. == checks whether two references point to the same object.

Null-first pattern: obj != null && obj.method()

compareTo meaning: negative means first < second, zero means equal, positive means first > second.

Most-tested reminders
Common trap When applying De Morgan's laws, flip the boolean operator and negate each condition. Example: !(x > 5 && y < 3) becomes x <= 5 || y >= 3.
Tracing tip Make a variable table, evaluate each condition left to right, and record which block runs.
Core rule Short-circuit evaluation is not just convenient. It prevents errors when the second part of a condition would be unsafe.
Unit 2B • 25-35%

Loops and iteration

This unit emphasizes loop structure, standard loop patterns, and common off-by-one issues.

while and for loops

while (cond) { body; update; } checks the condition before each iteration and may run zero times.

for (init; cond; update) { body; } runs initialization once, then cycles through condition, body, and update.

for-each: for (Type x : collection) { ... } is for reading, not indexed modification.

Infinite loops happen when the condition never becomes false, often because the update is missing or wrong.

Loop scope: a variable declared in a for header exists only inside that loop.

Standard loop patterns
PatternCode sketchTypical initialization
Count / Sumsum += arr[i]sum = 0
Searchif (== target) return iNone required
Filterif (cond) count++count = 0
Min / Maxif (arr[i] > max) max = valmax = arr[0]

Sentinel loop: a special value such as -1 signals when to stop.

Nested loops and complexity

Nested loops: if the outer loop runs N times and the inner loop runs M times, the inner body executes N × M times.

String traversal: for (int i = 0; i < str.length(); i++)

Reverse a string: loop from length() - 1 down to 0 and concatenate each character.

Informal runtime ideas: single loop → O(n), nested loops → O(n²), repeated halving → O(log n).

Most-tested reminders
Common trap i <= arr.length causes an ArrayIndexOutOfBoundsException. Use i < arr.length.
FRQ tip Standard patterns such as accumulate, search, and filter often align directly with rubric points.
Core rule When tracing a for loop, always think in the order: initialize, check, run body, update, repeat.
Unit 3 • 10-18%

Class creation

This is the class-design portion of the course and a direct FRQ target.

Anatomy of a class

Class: a blueprint. Object: an instance with its own state and behavior.

Instance variables: declared in the class body, outside all methods, usually private.

Constructor: public ClassName(params) { ... }. Same name as the class and no return type.

this: refers to the current object. Example: this.name = name;

No-arg constructor: Java provides one automatically only when you write zero constructors.

Methods and access
Modifier or kindMeaningTypical use
publicAccessible anywhereMethods in the public API
privateAccessible only in the classInstance variables
staticBelongs to the classUtilities and constants
voidNo return valueMutators / setters
non-voidReturns a valueAccessors / getters

Accessor: returns a field value.

Mutator: changes a field value.

toString(): returns a String representation and is used automatically by print() and String concatenation.

Scope, static, and design

Local variables exist only inside the method or block where they are declared and must be initialized before use.

Static members are shared across all instances and accessed through the class name.

Encapsulation means using private fields and public methods to protect state and control access.

Most-tested reminders
FRQ focus Class-design FRQs require private fields, constructor(s), all required methods, and exact return types.
Design tip Do not recompute values that are meant to be stored. Use instance variables inside methods.
Core idea The point of a class is to package state and behavior together in one object.
Unit 4A • 30-40%

Arrays and ArrayList

Students need to distinguish clearly between fixed-size arrays and dynamic ArrayLists.

Arrays

Declare: int[] arr = new int[n]; or int[] arr = {1, 2, 3};

Size: fixed after creation.

Index range: 0 through arr.length - 1

Last item: arr[arr.length - 1]

Defaults: int[] → 0, double[] → 0.0, boolean[] → false, String[] → null

Traversal: for (int i = 0; i < arr.length; i++)

ArrayIndexOutOfBoundsException: happens when the index is below 0 or greater than or equal to length.

ArrayList

Import: import java.util.ArrayList;

MethodWhat it doesReturns
add(obj)Append to the endboolean
add(i, obj)Insert at index i, shifting rightvoid
get(i)Retrieve element at index iE
set(i, obj)Replace the element at index iold E
remove(i)Remove at index i, shifting leftold E

Wrapper classes: use Integer instead of int in an ArrayList.

Autoboxing: Java converts between int and Integer automatically.

ArrayList vs array: resizable vs fixed, .get(i) vs [i], .size() vs .length, and .set(i, val) vs arr[i] = val.

Traversal pitfalls

Removing while going forward skips the next element because indices shift left.

Fixes: traverse backward, or do i-- immediately after a removal in a forward loop.

Enhanced for loops are good for reading, but not for removing items, modifying primitive elements, or using indices.

Concurrent modification issue: do not use for-each with removal. Use an indexed loop instead.

Most-tested reminders
Common trap Removing while traversing forward skips elements. Traverse backward: for (int i = size() - 1; i >= 0; i--).
FRQ tip Practice traversal, filtering, accumulation, and conditional removal. Those are standard ArrayList FRQ moves.
Core rule Know which methods shift elements left or right after insertion and removal.
Unit 4B

2D arrays and searching

This unit combines nested traversal with search patterns and careful bounds checking.

2D array basics

Declare: int[][] grid = new int[rows][cols]; or int[][] g = {{1, 2}, {3, 4}};

Access: grid[r][c]

Rows: grid.length

Columns: grid[0].length

Think of a 2D array as an array of arrays. Each grid[r] is a row.

Defaults: same as 1D arrays. For example, int[][] → 0 and String[][] → null.

Traversal orders

Row-major order: outer loop on rows, inner loop on columns. This is the most common exam order.

Column-major order: outer loop on columns, inner loop on rows.

For-each traversal: for (int[] row : grid) followed by for (int val : row)

Neighbor access: up grid[r - 1][c], down grid[r + 1][c], left grid[r][c - 1], right grid[r][c + 1]

Bounds first: check index validity before accessing a neighbor, and use short-circuit logic so unsafe expressions are never evaluated.

Searching algorithms and 2D patterns

Sequential / linear search: checks each element. Works on any data, sorted or not. Typical runtime O(n).

Binary search: requires sorted data. Use lo = 0, hi = len - 1, and mid = (lo + hi) / 2.

Binary logic: if target is less than the middle value, move hi left. If greater, move lo right. If equal, stop.

Binary search exit rule: the loop must continue while lo <= hi. If the pointers cross, the target is not present.

Common 2D patterns: row sum, column sum, counting matches, finding a max in each row, and working along the main diagonal where r == c.

Most-tested reminders
Common trap One bad index produces an ArrayIndexOutOfBoundsException. Check bounds before you touch neighbor cells.
FRQ tip Practice row traversal, column traversal, counting, min and max, and column-wise operations.
Core rule Binary search only works on sorted data. That condition matters every time.
Closing topics

Sorting, recursion, and FRQ strategy

These ideas often appear as tracing tasks, conceptual multiple-choice questions, or strategy notes for the free response section.

Sorting algorithms
SortHow it worksTime
SelectionFind the minimum and swap it to the frontO(n²)
InsertionShift the sorted part right and insert the next valueO(n²)
MergeSplit into halves, then merge sorted halves back togetherO(n log n)

Selection sort: makes exactly one swap per pass.

Insertion sort: is very fast on nearly sorted data.

Merge sort: is recursive, stable, and faster overall on large data sets.

Exam emphasis: trace the passes, identify the sort from behavior, and compare efficiency.

Recursion

Base case: stops the recursion and must be reachable.

Recursive case: calls the method again with a smaller or simpler input and must move toward the base case.

Tracing idea: draw the call stack and track local variables at each level.

Common examples: factorial, Fibonacci, powers, reversing a String, and binary search.

Error to avoid: missing or wrong base cases lead to StackOverflowError or incorrect results.

FRQ strategy

Q1: methods and control structures, especially loops and conditionals.

Q2: class design with fields, constructor(s), and methods.

Q3: ArrayList data analysis.

Q4: 2D array traversal.

Time target: about 22 minutes per question.

Best habits: read the full prompt, use the exact method signatures, and call all provided helper methods.

Most-tested reminders
Common trap Write the base case first. A missing base case causes a StackOverflowError.
FRQ tip Use the method signatures exactly as given, and do not ignore helper methods mentioned in the prompt.
Core rule For sorting and recursion questions, trace one step at a time. Precision matters more than speed.
